# 02 — User Journeys

One file per end-to-end journey a persona takes to reach a goal. Journeys give context that individual stories lack: the sequence of touchpoints, the pain points, and where the opportunities are.

## When to create one

When several stories cluster around the same persona and goal, or when a transcript describes a multi-step experience (especially a painful one) worth mapping.

## Naming

`UJ-###-short-kebab-title.md`, e.g. `UJ-001-locked-out-customer.md`.

Copy `TEMPLATE-user-journey.md`, rename, fill in.

## Quality checklist

- The journey has **one persona and one goal** — split it if it forks.
- Each stage names the **touchpoint** (login page, email, phone line…) — that's where design decisions land.
- Pain points come from **evidence in transcripts**, not guesswork; cite the transcript ID.
