# 01 — Transcripts

Records of requirements-elicitation conversations between the business analyst and stakeholders/users. These are the **raw source material** every downstream document traces back to.

## When to create one

One file per session — interviews, workshops, demos with feedback, or any conversation where requirements surfaced. Write it up the same day while memory is fresh.

## Naming

`TRN-###-short-kebab-topic.md`, e.g. `TRN-001-account-access-interview.md`.

Copy `TEMPLATE-transcript.md`, rename, fill in.

## Tips for good session records

- Capture **verbatim quotes** for anything contentious or surprising — paraphrasing loses nuance.
- Separate what was *said* (transcript) from what you *concluded* (key takeaways) — later readers must be able to tell them apart.
- Log **candidate requirements** as you spot them, even half-formed; they get formalised in `02-user-stories/` and `04-requirements/` later.
- Record open questions with an owner and follow-up date, or they will be forgotten.
